## v0.5.3

### Highlights (EN)

- New: per-client routing mode — `mode: auto | manual` with `pinned_provider` for manual (no-failover) routing.
- New: circuit breaker for each provider (`closed/open/half_open`) to avoid repeatedly hitting unhealthy upstreams.
- Improved: Web UI status now includes runtime details (mode/pin, last switch, per-provider deactivation + circuit state).
- Compatibility: provider `priority` is 1-based; if omitted (or `0`), it defaults to `1` (create/update still rejects `< 1`).
- Logging: `log_retention_days: 0` now means "keep forever" (no cleanup).

### 亮点 (中文)

- 新增：每个客户端支持路由模式 `mode: auto | manual`，`manual` 可通过 `pinned_provider` 锁定 provider（不再 failover）。
- 新增：Provider 级熔断器（`closed/open/half_open`），避免持续向不健康上游发送请求。
- 改进：Web 管理界面状态页增加运行态信息（mode/pin、上次切换、每个 provider 的禁用/熔断状态等）。
- 兼容：`priority` 从 `1` 开始；若配置中省略（或为 `0`）则默认按 `1` 处理（新增/更新仍拒绝 `< 1` 并提示）。
- 日志：`log_retention_days: 0` 表示永久保留（不清理历史日志）。
