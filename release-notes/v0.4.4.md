## v0.4.4

### Highlights (EN)

- Windows: `clipal service` now accepts flags both before and after the action (so `clipal.exe service install --dry-run` works as expected).
- Windows: Task Scheduler install is improved (run as the current user in the interactive session) and the task starts clipal with `--detach-console` so it stays running in the background.
- Windows: avoid mojibake from `schtasks.exe` output by inheriting the console handles instead of capturing and re-printing bytes.
- Logs: write to the log file before stdout, improving resilience when stdout is not available (common in background runs).
- Docs: update Windows guides (EN/ZH) with the new `clipal service` behavior and troubleshooting tips.

### 亮点 (中文)

- Windows：`clipal service` 支持 flags 放在 action 前或后（例如 `clipal.exe service install --dry-run` 可正常工作）。
- Windows：任务计划程序安装更稳（以当前用户/交互会话运行），并通过 `--detach-console` 让任务启动后在后台常驻运行。
- Windows：`schtasks.exe` 输出不再乱码（继承控制台句柄，避免抓取后重新打印导致的编码错乱）。
- 日志：优先写入日志文件，再写 stdout；在后台运行 stdout 不可用时更稳。
- 文档：更新 Windows 中英文指南，补充新行为说明与排错要点。
