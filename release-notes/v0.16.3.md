## v0.16.3

### English

#### Highlights

- Hardened OAuth authorization cleanup so cancelling the Web UI flow releases local callback listeners immediately.
- Added Codex native `auth.json` import support and simplified OAuth import to direct JSON file selection.
- Added terminal OAuth session cleanup to avoid retaining completed, failed, or expired login sessions indefinitely.

#### What Changed

- Fixed: cancelling an OAuth authorization from the Web UI now best-effort cancels the backend session and closes its local callback listener.
- Fixed: completed, failed, and expired OAuth login sessions are retained only for a short grace window before being cleaned up.
- Added: OAuth imports now recognize Codex CLI native `auth.json` files in addition to existing CLI Proxy API and `sub2api` formats.
- Improved: OAuth import UI now uses direct JSON file selection only, removing the ambiguous directory import flow.
- Improved: Codex OAuth identity parsing now also reads the email claim from the OpenAI profile namespace used by native access tokens.

#### Thanks

- Thanks to review and user feedback for tightening OAuth authorization lifecycle handling and import ergonomics.

### 中文

#### 亮点

- 加固 OAuth 授权清理流程，Web UI 取消授权后会立即释放本地 callback listener。
- 新增 Codex 原生 `auth.json` 导入支持，并将 OAuth 导入简化为直接选择 JSON 文件。
- 新增 OAuth 终态 session 清理，避免 completed / failed / expired 登录会话长期驻留内存。

#### 变更内容

- 修复：Web UI 取消 OAuth 授权时，会 best-effort 取消后端 session 并关闭本地 callback listener。
- 修复：completed / failed / expired OAuth 登录 session 只保留一个短暂宽限窗口，随后自动清理。
- 新增：OAuth 导入支持识别 Codex CLI 原生 `auth.json`，同时继续兼容 CLI Proxy API 与 `sub2api` 格式。
- 改进：OAuth 导入 UI 只保留直接选择 JSON 文件，移除容易产生歧义的目录导入流程。
- 改进：Codex OAuth 身份解析会同时读取原生 access token 使用的 OpenAI profile email claim。

#### 致谢

- 感谢 review 与用户反馈帮助收紧 OAuth 授权生命周期和导入体验。
