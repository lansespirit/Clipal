## v0.11.0

### English

#### Highlights

- Redesigned the Web UI `Global Settings` experience so routing, notifications, and core runtime controls are organized more clearly and take less vertical space.
- Refactored the Web UI styling system into layered CSS primitives and page/component styles, making the current interface more consistent and future UI work easier to maintain.
- Fixed a config consistency bug where clearing routing duration fields in the UI did not actually clear the saved backend value.

#### What Changed

- Added: a redesigned `Global Settings` page structure with denser layout, clearer grouping, and updated form coverage aligned with the current runtime configuration surface
- Improved: status and provider presentation in the Web UI so current, active, and disabled providers read more clearly and the surrounding badge / pill language is more consistent
- Improved: Web UI CSS architecture now loads through dedicated `tokens`, `base`, `primitives`, `components`, and `pages` layers instead of continuing to grow a single fallback stylesheet
- Added: `docs/frontend/web-ui-css-guidelines.md` to document the CSS layering model and naming discipline used by the refreshed Web UI
- Fixed: YAML export / API wiring for the newer global settings fields so the redesigned settings page and backend config stay aligned
- Fixed: clearing `routing.sticky_sessions.explicit_ttl`, `routing.busy_backpressure.short_retry_after_max`, or `routing.busy_backpressure.max_inline_wait` in the Web UI now persists correctly and falls back to runtime defaults as intended

### 中文

#### 亮点

- 重做了 Web UI 的 `Global Settings` 体验，让 routing、notifications 和核心运行参数的组织更清晰，整体更紧凑。
- 将 Web UI 样式系统重构为分层 CSS 架构，把当前界面的一致性和后续维护性一起补齐。
- 修复了一个前后端配置不一致问题：用户在界面里清空 routing duration 字段后，后端之前不会真正清空保存值。

#### 变更内容

- 新增：重新设计的 `Global Settings` 页面结构，布局更紧凑、分组更清晰，并且表单覆盖范围对齐到当前运行时配置能力
- 改进：Web UI 中的状态与 provider 展示方式，让 current、active、disabled 的信息分组更清楚，同时统一周边 badge / pill 的尺寸语言
- 改进：Web UI CSS 现已按 `tokens`、`base`、`primitives`、`components`、`pages` 分层加载，不再继续把样式堆进单一兜底样式表
- 新增：`docs/frontend/web-ui-css-guidelines.md`，用于说明新版 Web UI 的 CSS 分层模型与命名约束
- 修复：补齐新版 Global Settings 相关字段在 YAML 导出与 API 之间的对齐，避免前端页面与后端配置语义脱节
- 修复：在 Web UI 中清空 `routing.sticky_sessions.explicit_ttl`、`routing.busy_backpressure.short_retry_after_max`、`routing.busy_backpressure.max_inline_wait` 后，现在会正确持久化为空值，并按预期回退到运行时默认值
