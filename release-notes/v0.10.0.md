## v0.10.0

### English

#### Highlights

- Added `CLI Takeover` to the Web UI so Clipal can apply, preview, back up, and roll back user-level config takeover for supported local AI coding clients.
- Expanded one-click takeover beyond Claude Code and Codex CLI to cover `OpenCode`, `Gemini CLI`, `Continue`, `Aider`, and `Goose`.
- Tightened takeover safety around real-world client behavior, including Claude Code onboarding state handling and product-specific warnings about higher-precedence config layers.

#### What Changed

- Added: Web UI `CLI Takeover` tab with per-client status, target path display, current-vs-planned file preview, apply, and rollback
- Added: managed takeover support for `~/.claude/settings.json`
- Added: managed takeover support for `~/.codex/config.toml`
- Added: managed takeover support for `~/.config/opencode/opencode.json`
- Added: managed takeover support for `~/.gemini/.env`
- Added: managed takeover support for `~/.continue/config.yaml`
- Added: managed takeover support for `~/.aider.conf.yml`
- Added: managed takeover support for `~/.config/goose/custom_providers/clipal.json`
- Added: per-product backup manifests stored under Clipal's config directory so rollback restores the last captured pre-takeover state
- Improved: Claude Code takeover now also ensures `~/.claude.json` records `hasCompletedOnboarding: true` so the user-level base URL override can take effect immediately after takeover
- Improved: Codex takeover now writes the dedicated `[model_providers.clipal]` section without the extra parent `[model_providers]` table
- Improved: Gemini CLI takeover only updates `GEMINI_API_BASE`, leaving the rest of the user `.env` intact
- Improved: Aider takeover preserves existing keys and fills a placeholder `openai-api-key` only when no key is present
- Improved: UI warnings now explain when workspace, project-local, environment, managed, or in-app selection layers may still override the user-level takeover
- Improved: integration and Web regression coverage for preview/apply/rollback flows across all supported takeover targets

### 中文

#### 亮点

- Web UI 新增 `CLI Takeover`，可对受支持的本地 AI 编码客户端执行一键接管，并在 apply 前预览、自动备份、支持 rollback。
- 一键接管能力从 Claude Code 和 Codex CLI 扩展到 `OpenCode`、`Gemini CLI`、`Continue`、`Aider`、`Goose`。
- 针对真实客户端行为补齐了更多安全细节，例如 Claude Code 的 onboarding 状态处理，以及各产品更明确的高优先级覆盖提示。

#### 变更内容

- 新增：Web UI `CLI Takeover` 标签页，支持按客户端展示状态、目标路径、当前文件与计划写入内容，并提供 apply / rollback
- 新增：对 `~/.claude/settings.json` 的受管接管支持
- 新增：对 `~/.codex/config.toml` 的受管接管支持
- 新增：对 `~/.config/opencode/opencode.json` 的受管接管支持
- 新增：对 `~/.gemini/.env` 的受管接管支持
- 新增：对 `~/.continue/config.yaml` 的受管接管支持
- 新增：对 `~/.aider.conf.yml` 的受管接管支持
- 新增：对 `~/.config/goose/custom_providers/clipal.json` 的受管接管支持
- 新增：按产品保存的 takeover 备份与 manifest，存放在 Clipal 配置目录下，rollback 可恢复最近一次接管前状态
- 改进：Claude Code 接管时会同步确保 `~/.claude.json` 中的 `hasCompletedOnboarding: true`，让用户级 Base URL 覆盖能在接管后立即生效
- 改进：Codex 接管现在直接写专用的 `[model_providers.clipal]` 段，不再生成多余的父级 `[model_providers]`
- 改进：Gemini CLI 接管只更新 `GEMINI_API_BASE`，其余用户 `.env` 内容保持不动
- 改进：Aider 接管会保留已有 key，并且只在缺失时补一个占位 `openai-api-key`
- 改进：UI 警告文案会明确说明 workspace、项目级配置、环境变量、受管策略或应用内当前选择仍可能覆盖用户级接管结果
- 改进：补齐各受支持接管目标在 preview / apply / rollback 上的 integration 与 Web 回归测试
