## v0.6.1

### English

#### Highlights

- Refined the Add Provider and Edit Provider dialogs in the Web UI for multi-key providers.
- Improved spacing, field hierarchy, and helper text so the provider form is easier to scan.
- Made the API key textarea clearer by separating input guidance from storage behavior.
- Expanded the modal layout for better readability on desktop while keeping mobile actions comfortable.

#### What Changed

- Improved: clearer form layout and copy for provider create/edit dialogs
- Improved: dedicated multi-line textarea styling for API key entry
- Improved: edit-mode guidance now explicitly explains that leaving API keys empty keeps the existing keys
- Improved: modal sizing and responsive footer behavior for smaller screens

### 中文

#### 亮点

- 优化了 Web UI 中 Add Provider / Edit Provider 弹窗，提升多 key provider 的录入体验。
- 改进了字段间距、信息层级和辅助文案，让表单更容易扫读。
- 将 API Key 输入说明与保存行为拆开显示，多行输入语义更清楚。
- 扩展了桌面端弹窗宽度，并优化了移动端底部操作区的排版。

- 改进：优化 provider 新增/编辑弹窗的表单布局与文案
- 改进：为 API Key 多行输入增加专门的 textarea 样式
- 改进：编辑态明确说明留空会保留现有 key
