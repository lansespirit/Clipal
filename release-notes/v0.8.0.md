## v0.8.0

### English

#### Highlights

- Added session-sticky busy overflow routing for OpenAI-, Claude-, and Gemini-style traffic.
- Concurrency-limit `429` responses are now treated as short-lived busy signals instead of immediately poisoning provider health.
- Clipal can now keep multi-turn traffic on the same provider using explicit response chains, cache hints, and bounded human-message features.
- Added routing config knobs for sticky affinity TTLs, dynamic feature cache size, inline wait backoff, and busy probe limits.

#### What Changed

- Added: request-side sticky affinity for OpenAI `previous_response_id`, `prompt_cache_key`, Anthropic `container`, and Gemini `cached_content`
- Added: short-lived dynamic conversation affinity for stateless multi-turn message APIs using hashed human-message features
- Added: bounded inline wait and retry probe before overflowing busy sessions to the next provider
- Added: sticky rebinding after successful busy overflow so later requests keep using the overflow target
- Added: routing config in `config.yaml` for `sticky_sessions` and `busy_backpressure`
- Improved: hot reload now preserves compatible sticky affinity, response lookup, and busy runtime state
- Improved: routing tests now cover busy classification, sticky learning, previous-response chaining, first-turn learning, and bounded cache eviction
- Improved: config reference and example config now document the new routing controls

### 中文

#### 亮点

- 新增 session-sticky busy overflow routing，面向 OpenAI、Claude、Gemini 风格流量提供更智能的黏性路由。
- 并发限制类 `429` 现在会被视为短期 busy 信号，而不是立刻把 provider 判成不健康。
- Clipal 现在可以基于显式 response 链路、缓存 hint 与受限的人类消息特征，让多轮请求尽量保持在同一个 provider 上。
- 新增路由配置项，可控制黏性 TTL、动态特征缓存容量、inline wait/backoff 和 busy 探测并发。

#### 变更内容

- 新增：为 OpenAI `previous_response_id`、`prompt_cache_key`、Anthropic `container` 与 Gemini `cached_content` 提供请求侧黏性路由
- 新增：为无显式会话 ID 的多轮消息接口提供短期动态对话黏性，基于哈希化的人类消息特征
- 新增：provider busy 时先进行 bounded inline wait 与恢复探测，失败后再 overflow 到下一个 provider
- 新增：busy overflow 成功后会重绑黏性会话，后续请求继续固定到新的 provider
- 新增：`config.yaml` 中的 `sticky_sessions` 与 `busy_backpressure` 路由配置
- 改进：热重载现在会在兼容条件下保留 sticky affinity、response lookup 与 busy 运行态
- 改进：路由测试补齐了 busy 分类、sticky 学习、`previous_response_id` 链路、首轮学习以及有界缓存淘汰
- 改进：配置参考和示例配置文件现已包含新的路由控制项
