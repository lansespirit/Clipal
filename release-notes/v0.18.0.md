## v0.18.0

### English

#### Highlights

- Synced the Claude Code OAuth wrapper with current Claude Code traffic.
- Hardened Codex OAuth request and response compatibility.
- Fixed active-provider status reporting for manual routing mode.

#### What Changed

- Fixed: Claude OAuth requests now synthesize the Claude Code-compatible system, tool, metadata, cache-control, context-management, and billing/signature envelope expected by current Claude Code releases.
- Fixed: Claude OAuth wrapping now preserves client-supplied `system` and `tools` instead of replacing them with defaults; defaults are only added when the client omits those fields.
- Fixed: Claude OAuth request attribution now derives billing fingerprints from the real user prompt and avoids letting injected helper prompts pollute attribution.
- Fixed: Claude OAuth signing now uses the seeded hash behavior and only signs the billing headers that Claude Code signs.
- Fixed: Claude OAuth default tool schemas were expanded for closer `Agent` and `Bash` compatibility.
- Improved: Codex OAuth wrapping now better matches current local Codex behavior for model defaults, provider overrides, and response handling.
- Fixed: `clipal status` now reports the active provider correctly when a client is in manual mode.
- Improved: Added regression coverage and a live Claude OAuth smoke path for the updated wrapper behavior.

#### Thanks

- Thanks to the subagent audits, local Claude Code verification, and live `3433` testing that caught the provider override and model-version edge cases before release.

### 中文

#### 亮点

- 将 Claude Code OAuth 包装同步到当前 Claude Code 流量形态。
- 加固 Codex OAuth 请求与响应兼容性。
- 修复 manual 路由模式下 active provider 状态展示。

#### 变更内容

- 修复：Claude OAuth 请求现在会合成当前 Claude Code 需要的 system、tool、metadata、cache-control、context-management、billing/signature envelope。
- 修复：Claude OAuth 包装现在会保留客户端显式传入的 `system` 和 `tools`；只有客户端缺省时才补默认值。
- 修复：Claude OAuth 请求归因现在基于真实用户提示词生成 billing fingerprint，避免注入的辅助提示污染归因。
- 修复：Claude OAuth 签名改为使用 seeded hash 行为，并且只签 Claude Code 会签的 billing headers。
- 修复：补齐 Claude OAuth 默认工具 schema，使 `Agent` 和 `Bash` 更贴近官方能力。
- 改进：Codex OAuth 包装进一步对齐当前本地 Codex 的模型默认值、provider override 和响应处理行为。
- 修复：`clipal status` 现在会在 client 使用 manual 模式时正确报告当前 active provider。
- 改进：新增回归覆盖，并补充用于新版包装行为的 Claude OAuth live smoke 路径。

#### 致谢

- 感谢 subagent 审计、本地 Claude Code 核对和 `3433` live 测试，提前发现 provider override 与模型版本边界问题。
