## v0.17.2

### English

#### Highlights

- Refined provider usage cards in the Web UI so reasoning and thoughts details move into hover tooltips instead of overflowing the card layout.
- Fixed API-key provider cost inference so usage-based spend can be computed from official OpenAI, Claude, and Gemini response fields rather than working only for OAuth-backed providers.
- Reduced tooltip noise on provider cards by suppressing the drag-reorder hint while a usage or spend tooltip is active.

#### What Changed

- Improved: provider usage and input/output rows now keep the compact primary token figures visible and move detailed reasoning or thoughts breakdowns into explicit in-page tooltips.
- Fixed: usage extraction now recognizes additional official SDK field shapes, including OpenAI `completion_tokens_details.reasoning_tokens` and Gemini `responseTokenCount` plus `toolUsePromptTokenCount`.
- Fixed: spend estimation now runs for API-key providers across supported protocol families when the request model and usage payload provide enough information.
- Improved: provider-card hover behavior now avoids showing the drag hint at the same time as usage or spend tooltips.

#### Thanks

- Thanks to the live Web UI verification that caught the tooltip overlap and API-provider spend visibility gaps before release.

### 中文

#### 亮点

- 优化 Web UI 中 provider usage 卡片：reasoning / thoughts 细项改为悬停提示展示，不再把卡片内容撑爆。
- 修复 API key provider 的成本推导逻辑：现在可以基于 OpenAI、Claude、Gemini 官方响应字段计算 spend，而不再只有 OAuth provider 能正常显示金额。
- 降低 tooltip 干扰：当 usage 或 spend tooltip 显示时，会临时隐藏拖拽排序提示。

#### 变更内容

- 改进：provider usage 与 input/output 行保留紧凑主数值，reasoning / thoughts 等详细拆分移入页面内 tooltip。
- 修复：usage 提取逻辑补充兼容更多官方 SDK 字段形态，包括 OpenAI 的 `completion_tokens_details.reasoning_tokens`，以及 Gemini 的 `responseTokenCount` 和 `toolUsePromptTokenCount`。
- 修复：在请求模型和 usage 数据足够时，API key provider 现在也会进行 spend 估算并显示金额。
- 改进：provider 卡片悬停行为调整为 usage / spend tooltip 优先，不再与拖拽提示同时出现。

#### 致谢

- 感谢这次针对 Web UI 的现场验证，及时发现并补齐了 tooltip 重叠和 API provider 金额可见性问题。
