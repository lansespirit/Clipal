## v0.16.2

### English

#### Highlights

- Fixed OAuth provider proxy settings so proxy choices made during authorization are preserved on the linked account and used for later token refreshes.
- Improved proxy request hot paths to avoid unnecessary large body reads, repeated JSON rewriting, synchronous telemetry fsyncs, and over-flushing non-streaming responses.
- Preserved Claude OAuth's specialized transport for default and direct connections while still honoring explicit custom proxy settings.

#### What Changed

- Fixed: OAuth authorization now carries provider proxy settings from the Web UI start request through session polling and final provider linking.
- Fixed: OAuth token refresh and Codex usage metadata requests now use the effective provider/global upstream proxy policy.
- Fixed: default provider proxy mode is no longer treated as an explicit proxy override during OAuth authorization.
- Fixed: Claude OAuth direct/default flows preserve the Claude-specific uTLS transport instead of replacing it with a generic HTTP client.
- Improved: failover now checks provider/key/circuit availability before reading large request bodies.
- Improved: request handling now uses a per-request payload envelope with lazy JSON parsing and provider/OAuth rewrite caches.
- Improved: telemetry persistence now runs through a background throttled writer, with a forced flush on shutdown.
- Improved: non-SSE responses no longer flush after every chunk; streaming responses still flush for low latency.

#### Thanks

- Thanks to review feedback for identifying OAuth proxy propagation gaps and hot-path performance regressions.

### 中文

#### 亮点

- 修复 OAuth provider 代理设置链路，授权时选择的代理会保存在成功关联的账号 provider 上，并用于后续 token refresh。
- 优化代理请求热路径，避免无可用上游时读取大 body、重复 JSON 重写、请求完成路径同步 telemetry fsync，以及非流式响应过度 flush。
- 保留 Claude OAuth 默认 / direct 连接下的专用 transport，同时继续支持显式 custom proxy。

#### 变更内容

- 修复：OAuth 授权会从 Web UI start 请求开始携带 provider 代理设置，并贯穿 session poll 和最终 provider link。
- 修复：OAuth token refresh 与 Codex usage metadata 请求会使用当前 provider/global 的有效上游代理策略。
- 修复：`proxy_mode=default` 不再被视为 OAuth 授权中的显式代理覆盖。
- 修复：Claude OAuth default/direct 流程保留 Claude 专用 uTLS transport，不再被通用 HTTP client 替换。
- 改进：failover 会先检查 provider/key/circuit 可用性，再读取大请求体。
- 改进：请求处理引入 per-request payload envelope，支持 lazy JSON parse 与 provider/OAuth rewrite 缓存。
- 改进：telemetry 持久化改为后台节流写入，并在 shutdown 时强制 flush。
- 改进：非 SSE 响应不再每个 chunk flush；流式响应仍保持低延迟 flush。

#### 致谢

- 感谢 review 反馈帮助定位 OAuth 代理传递缺口和热路径性能回归。
