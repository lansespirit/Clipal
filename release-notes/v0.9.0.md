## v0.9.0

### English

#### Highlights

- Canonicalized Clipal's client naming around `claude`, `openai`, and `gemini` across config files, Web UI, runtime identifiers, and API output.
- Added one-time startup migration from legacy config filenames to `claude.yaml` and `openai.yaml`.
- Brought the bundled Clipal icon into the Web UI header and favicon so the product branding is no longer README-only.

#### What Changed

- Added: startup config migration from `claude-code.yaml` to `claude.yaml` and from `codex.yaml` to `openai.yaml`
- Added: conflict detection when both legacy and canonical config filenames exist with different content
- Changed: Web API status/export payloads now use canonical client keys `claude`, `openai`, and `gemini`
- Changed: internal config/runtime identifiers now use canonical `Claude`, `OpenAI`, and `Gemini` naming
- Changed: canonical compatibility aliases `/claude` and `/openai` are now available alongside `/clipal`
- Improved: Web UI client tabs and labels now show `Claude`, `OpenAI`, and `Gemini`
- Improved: `clipal status` uses canonical user-facing client names
- Improved: docs, examples, smoke tests, and fixtures now reference `claude.yaml` and `openai.yaml`

### 中文

#### 亮点

- 将 Clipal 的客户端命名统一为 `claude`、`openai`、`gemini`，覆盖配置文件、Web UI、运行时标识与 API 输出。
- 新增启动时的一次性配置文件迁移：自动把旧文件名迁移到 `claude.yaml` 和 `openai.yaml`。
- Web UI 现已接入内置 Clipal 图标与 favicon，品牌资产不再只出现在 README 中。

#### 变更内容

- 新增：启动时自动将 `claude-code.yaml` 迁移为 `claude.yaml`，将 `codex.yaml` 迁移为 `openai.yaml`
- 新增：当旧文件名与新文件名同时存在且内容不一致时，直接报错阻止歧义加载
- 调整：Web API 的状态与导出 JSON 现在统一输出 `claude`、`openai`、`gemini` 三个 canonical key
- 调整：内部配置与运行时标识统一改为 `Claude`、`OpenAI`、`Gemini`
- 调整：除 `/clipal` 外，新增 canonical 兼容入口 `/claude` 与 `/openai`
- 改进：Web UI 客户端标签统一显示为 `Claude`、`OpenAI`、`Gemini`
- 改进：`clipal status` 现在使用统一后的用户可见命名
- 改进：文档、示例、smoke test 与测试夹具统一改为 `claude.yaml` 和 `openai.yaml`
