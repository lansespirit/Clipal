## v0.10.1

### English

#### Highlights

- Refined the Web UI `Services` controls so service actions expose clearer state, safer disabled behavior, and a cleaner responsive layout.
- Aligned the GitHub release workflow with the repository's `build/`-based build pipeline and restored the missing Windows ARM64 release artifact.

#### What Changed

- Improved: the `Services` tab now distinguishes installed, loaded, and running state instead of collapsing everything into a single coarse `ok` flag
- Improved: service actions now use disabled reasons, busy state handling, and clearer runtime-aware primary actions in the Web UI
- Improved: service controls now use a desktop left/right action layout with a narrow-screen fallback
- Fixed: GitHub release workflow now builds via `scripts/build.sh` and publishes assets from `build/` instead of the old transient `dist/` directory
- Fixed: `scripts/build.sh` now also builds the `windows/arm64` binary so local builds and release assets stay aligned
- Fixed: release checksum generation now excludes `checksums.txt` itself, so published hashes only cover the actual binaries

### 中文

#### 亮点

- 继续打磨 Web UI 的 `Services` 控制区，让服务状态更清晰、disabled 行为更安全、布局在桌面和窄屏下都更稳定。
- 把 GitHub Release 流程对齐到仓库当前的 `build/` 构建链路，并补回之前本地构建脚本缺失的 Windows ARM64 发布产物。

#### 变更内容

- 改进：`Services` 标签页现在会区分 installed、loaded、running 状态，不再只暴露一个过于粗粒度的 `ok` 标记
- 改进：服务操作按钮现在具备 disabled reason、busy 态处理，以及基于当前运行态的主动作强调
- 改进：服务控制区采用桌面端左右分区、窄屏自动回落的响应式布局
- 修复：GitHub release workflow 现在通过 `scripts/build.sh` 构建，并从 `build/` 发布资产，不再继续使用旧的临时 `dist/` 目录
- 修复：`scripts/build.sh` 现已补上 `windows/arm64` 构建目标，确保本地构建与发布资产集合一致
- 修复：release 的 checksum 生成现在会排除 `checksums.txt` 自身，发布出的哈希只覆盖实际二进制文件
